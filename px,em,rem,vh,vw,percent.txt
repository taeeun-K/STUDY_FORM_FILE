✅ 1) px (픽셀 – 고정값)
✔ 특징
가장 안정적이고 변하지 않는 절대 단위
디자인 시안 그대로 구현할 때 정확함
반응형에 약함

✔ 쓰는 곳
border(1px, 2px)
icon 사이즈
box-shadow
아주 작은 padding, margin 등

✔ 기억 포인트
“픽셀 단위로 변하면 안 되는 요소는 px”

-------------------------------------------------------------------------------

✅ 2) em (부모 font-size 기준 상대 단위)
✔ 특징
부모 요소 폰트 크기에 따라 크기가 변함
중첩되면 커짐 (상속 영향 큼) . . . 의도하지않게 크기가 커지거나 작아질 수 있음.

예:
부모 font-size: 20px
→ 1em = 20px
→ 2em = 40px

✔ 쓰는 곳
버튼/폼 내부 padding
컴포넌트 내부에서 글자 크기에 비례해야 할 때

✔ 기억 포인트
“컴포넌트 내부 요소가 글자 크기와 함께 같이 커져야 하면 em”

-------------------------------------------------------------------------------

✅ 3) rem (root em — 루트(html) 기준)
✔ 의미
html 태그의 font-size를 기준으로 계산되는 단위.

기본적으로
html { font-size: 16px }
→ 1rem = 16px

✔ 특징
부모 영향 X → 전역적으로 안정적
반응형과 접근성에서 가장 선호되는 단위
사용자 브라우저 기본 폰트 크기 적용도 자연스러움

✔ 쓰는 곳
전체 레이아웃 패딩/마진
폰트 크기
전역 spacing 시스템
반응형 미세 조절

✔ 실무 꿀팁
html font-size를 10px로 세팅하면 계산 편함:
html { font-size: 62.5%; }  /* 16px의 62.5% = 10px */

→ 1rem = 10px
→ 계산 편리 (1.6rem → 16px)

✔ 기억 포인트
“사이트 전체 스케일이 조절되길 원하면 rem”

-------------------------------------------------------------------------------

✅ 4) vh (viewport height)
✔ 의미
화면 높이를 기준으로 % 계산.
1vh = 화면 높이의 1%
100vh = 화면 전체 높이

✔ 특징
화면 세로 길이에 따라 유동적
모바일 브라우저 주소창 높이 문제(vh 버그) 항상 주의해야 함

✔ 쓰는 곳
풀스크린 섹션 (메인배너, 랜딩 화면)
Vertical centering
섹션 전체 높이 맞출때
카드, 슬라이드 높이 맞출 때

✔ 기억 포인트
“세로 화면 채우기 = vh”

-------------------------------------------------------------------------------

✅ 5) vw (viewport width)
✔ 의미
화면 너비 % 기준.
1vw = 화면 가로의 1%
100vw = 화면 전체 너비

✔ 특징
반응형에서 width를 유동적으로 만들 때 매우 유용
텍스트도 vw로 하면 화면에 따라 자동 줄어듦

✔ 쓰는 곳
반응형 타이포그래피(텍스트)
배너 텍스트 크기
가로폭 100% UI
슬라이더/컨테이너/배경

✔ 기억 포인트
“가로 기반 유동 사이즈 = vw”

-------------------------------------------------------------------------------

✅ 6) % (부모 요소 기준 비율 단위)
✔ 특징
부모 요소의 크기를 기준으로 비율 계산
width: % → 가장 많이 사용
height: % → 부모가 height를 갖고 있어야 동작함

✔ 쓰는 곳
레이아웃 width 분배
box를 비율로 배치할 때
grid/gap 기반의 반응형 레이아웃

✔ 기억 포인트
“부모 요소 크기에 맞춰서 비율로 잡고 싶으면 %”

🎯 실무에서 가장 좋은 조합
💡 rem + px + vh/vw 조합

예:
폰트 = rem
레이아웃 padding = rem
버튼 내부 padding = em
border = px
전체 화면 배너 높이 = vh
반응형 텍스트 = clamp() + vw
실제로 최신 반응형 UI에서는 이렇게 씀.

✅ 단일 단위(rem만, vw만 등)로 ‘통일해서 쓰는 방식’은 오히려 비효율적이다.
✅ 실무에서는 px + rem + vh + vw 를 “용도에 맞게 섞어서” 쓰는 게 정석이다.

🔍 왜 한 가지 단위로 통일하면 안 되는가?
1) 각 단위가 설계된 목적 자체가 다르기 때문
px → 고정값 / UI 안정성
rem → 전역 스케일링 / 접근성
vh → 화면 높이 기반 연출
vw → 반응형 너비 기반 타이포그래피

⚠ rem로 height 조절 못함
⚠ vh로 font-size를 통일할 수 없음
⚠ vw로 padding까지 모두 잡으면 망함 (너비 변화에 따라 계속 움직임)

따라서, 각 단위의 역할이 서로 다름.

🧩 실무 기준: 가장 좋은 단위 조합 패턴
아래는 실제 기업·에이전시 프로젝트에서 쓰는 표준 패턴이야.
🔸 1) 폰트는 rem
접근성
사용자 브라우저 폰트 확대 시 자연스러움
반응형에서 전역적으로 관리 쉬움

🔸 2) border, 아이콘 등은 px
1px border는 변하면 안 됨
box-shadow 값, hairline 등은 px 고정이 정답

🔸 3) layout spacing(여백), padding/margin은 rem
레이아웃의 전체적인 비율 유지
root font-size를 조절하면 전체 간격이 스케일링됨

🔸 4) 화면 전체를 채워야 하는 영역 높이 = vh
예:
메인 비주얼 배너 100vh
풀스크린 섹션

🔸 5) 반응형 타이포그래피 / 비주얼적인 크기 = vw
예:
랜딩 페이지 큰 헤드라인
배너 타이틀
홈페이지 Hero 영역의 문구

단위변환 공식
px → rem	px / root-font-size
rem → px	rem × root-font-size
px → em	px / 부모 font-size
em → px	em × 부모 font-size
px → vw	(px / 화면 너비) × 100
vw → px	(vw × 화면 너비) / 100
px → vh	(px / 화면 높이) × 100
vh → px	(vh × 화면 높이) / 100
px → %	(자식 px / 부모 px) × 100
% → px	(비율 × 부모 px) / 100
