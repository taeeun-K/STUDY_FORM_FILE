상황 role을 써야 하는 경우 role을 쓰면 안 되는 경우
1. div/span만으로 UI를 만들 때 실제 의미를 부여하기 위해 필요
예: <div role="button"> 		div인데 실제 버튼 기능도 없고 단순 레이아웃만 담당할 경우
2. HTML 시맨틱 태그가 없는 경우 목록 UI인데 <div>로 만들었다면 role="list" / role="listitem" 시맨틱 태그(<ul>, <li>)를 쓸 수 있는데도 div로 억지로 role만 붙임
3. 모달/팝업 만들 때 <div role="dialog"> 필수
+ aria-labelledby, aria-describedby 		modal인데 role="dialog" 안 넣고 div만 사용하면 ❌
4. 탭/아코디언 같은 인터랙션 UI tab, tablist, tabpanel 사용 HTML만으로 시맨틱 구조 만들 수 없다고 해도
role을 엉뚱하게 사용하는 경우
5. Landmark 지정이 필요한 큰 영역 header → role="banner"
nav → role="navigation"
main → role="main" 		이미 <nav>, <main> 등 시맨틱 태그 쓰고 있으면
role 중복 필요 없음
6. 스크린리더가 자동으로 읽어야 하는 영역 에러 메시지, 알림 → role="alert" 일반 텍스트나 설명문에 role="alert" 쓰면
스크린리더가 계속 끼어들어 혼란 발생
7. section에 제목이 존재하고 영역 구분 필요 <section> → role="region" + aria-label 제목이 없거나 의미 없는 구역인데 괜히 role="region" 붙이면 안 됨
8. 단순 아이콘 시각적 장식 아이콘은 aria-hidden="true" 아이콘에 의미가 있는데 aria-hidden 넣어버리면 ❌
(예: 검색버튼 아이콘)
9. table 요소 div 기반 그리드 UI인데 스크린리더에서 테이블처럼 읽혀야 할 때
role="table" 		실제 <table> 쓰는 상황 + role 중복(필요 없음)
10. 입력 영역 custom checkbox, switch 같은 UI →
role="checkbox", role="switch" 		기본 <input type="checkbox">에는 role 넣으면 안 됨
(이미 의미 충분)
11. 메뉴 UI 드롭다운 메뉴 → role="menu", role="menuitem" 네비게이션 메뉴는 일반적으로 menu role 쓰지 않음
(오해의 소지 있음)
12. 단순한 텍스트/레이아웃 요소 role="presentation" 또는 role="none"으로 의미 제거 의미 있는 요소(<h1>, <h2>)인데
presentation 넣으면 스크린리더가 인식 못함
